# Contribution Guide

## Workflow

1. Clone the repository.
2. Checkout your assigned branch.
3. Develop your assigned feature.
4. Commit your changes.
5. Push your branch.
6. Open a Pull Request.
7. Wait for review before merging.

---

## Branches

- main
- frontend
- authentication
- scanner
- documentation

---

## Commit Message Convention

```
feat:
fix:
docs:
refactor:
style:
test:
```

Example:

```
feat: implement landing page
```

---

## Pull Requests

- Keep PRs focused on a single feature or issue.
- Link the related GitHub Issue.
- Ensure code compiles successfully.
- Request at least one review before merging.
